# FishMach 自动钓鱼机

- 作者: 羽学
- 出处: TShock官方群816771079
- 这是一个Tshock服务器插件，主要用于：
- 使用/afm 指令指定一个箱子作为具备移动物品动画的自动钓鱼机,支持自定义渔获物品与怪物

## 指令

| 语法           |        权限         |   说明   |
| -------------- | :-----------------: | :------: |
| /afm set | afm.use | 打开箱子设为钓鱼机 |
| /afm exc | afm.use | 打开箱子添加/移除手上排除物品 |
| /afm info | afm.use | 打开箱子获取钓鱼机信息 |
| /afm list | afm.use | 列出所有自钓机 |
| /afm loot | afm.use | 列出自定义渔获 |
| /afm cond | afm.admin | 查看自定渔获条件 |
| /afm npc | afm.admin | 修改自定渔获怪物 |
| /afm item | afm.admin | 修改自定渔获物品 |
| /afm reset | afm.admin | 重置插件数据 |
| /reload | tshock.cfg.reload | 重新加载配置文件 |

## 使用方法
- 放个箱子,使用/afm s指令,再打开箱子
- 给箱子放入鱼竿和鱼饵
- 如果开启【需要电路】,则需要连上电线与计时器
- 挖掉箱子可自动销毁钓鱼机

0: 仅阻止同类型怪物重复, 1: 阻止所有自定义怪物共存
钓出怪物只需要在搜索范围内就生效，不用打开箱子

移除了save子命令，更新了指令逻辑：
当使用set、info、exc命令时只需打开箱子即可生效
修复了保存时箱子是否已被其他钓鱼机使用
修复了保存时对附近钓鱼机的范围检查(避免共用一套渔具、鱼饵)
优化性能,避免每次钓Npc时都会遍历1次所有在线玩家
钓出怪物必须条件为:玩家打开过1次箱子才会生效(用于计算位置)
修复了物品动画,锚点位不再是从左往右而是取最近值
加入了【需要电路】配置项,该配置开启时：
- 玩家正在开箱子无法自动钓鱼(没连电线则提醒)
- 频率与警告广播全由电路决定
- 不再遍历所有机器(性能最佳)

v1.0.3
再次修复保存自动钓鱼机时,丢失箱子索引的问题
移除了自动搜索的找空箱存物品逻辑：
- 箱子满后不再从附近找空箱存物品
- 无空槽位则只钓可堆叠的物品
- 查找鱼竿和鱼饵与其他物品不受影响
加入了延时缓存与修改箱子物品事件刷新缓存：
实现大幅度优化代码性能,避免了多次实时计算带来的CPU压力
加入了新配置项【保存间隔帧数】、【刷新缓存秒数】
每次修改箱子物品时可同步刷新自钓机环境识别：
以修复了环境改变后钓鱼物品仍钓出保存时的环境物品

给自定义渔获加入了生成怪物功能,以便钓出血月怪
加入了新的配置项：
【警告广播】【钓任务鱼】【允许钓出怪物】【禁钓已有怪物】
自定义渔获表加入了【怪物ID】【自动描述】
自动描述无需自己填写,每次/reload后,由插件自动重写
优化了/afm loot指令,支持显示自动描述
重构自定渔获逻辑，当成功钓出时不再执行原版钓鱼规则
【警告广播】用于独立控制：缺失鱼竿和鱼饵、自定渔获播报
新增/afm npc指令,用于修改自定义渔获怪物
血月钓竿可以使自定渔获怪物产出概率翻倍

v1.0.1
新增功能：自定义额外渔获(对应指令loot,cond,item)
新增排除指令：/afm exc
- 需在游戏内用s指令敲击自钓箱,仅拥有者和管理可用
- 根据手持物品来决定,存在则移除,不在则添加
性能优化：渐进式搜索（从2格逐步扩大），优先命中钓鱼箱，减少无效扫描。
动态查找物品：鱼竿、鱼饵、加成物品（宝箱药水、钓具箱等）不再存储，改为实时扫描附近箱子，灵活生效。

智能物品存放：优先放入钓鱼箱，满则自动寻找最近的其他箱子，并播放原版转移动画。
宝箱概率可调：新增配置项 宝箱概率加成，支持自定义板条箱掉落率；宝箱药水效果同步生效。
箱子变动简化：只有自定义加成物品变动时才保存数据，鱼竿、鱼饵变动只提示，不再误清机器状态。

v1.0.0
实现让箱子具备移动物品动画的自动钓鱼机
